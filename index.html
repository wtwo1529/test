Recently, I tackled a challenging problem where I had to iterate over an array of integers and return a 2D array containing arrays of distinct integers from the initial array. To solve this problem, I iterated over the initial array from right to left and passed the integers into a hashmap that would reset after each iteration. This would allow for a conditional that could check if the iterable was in the hashmap, and then either continue to the next element or continue, append the element to a local list, and pop the element. After the iteration, the local list would be appended to a global list. This process would repeat until the initial array was empty. After solving the problem, I wanted to find out ways I could improve the time complexity of my solution. Through research, I found that a set could be used instead of a dictionary as it could act as a hashmap through the fact that it had O(1) look-up time, but only had keys instead of key-value pairs. Having only a key meant greater efficiency as I only needed to check if the iterable was in the hash map.
